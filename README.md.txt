Conceptos de diseño de software
Cuando uno escribe código este debe ser entendible por otros programadores.
Archivo.- 01_entendible.php

El código tiene que ser simple, eliminando exceso de complejidad.
Archivo.- 02_simple.php

Tiene que haber flexibilidad, no hacer especulaciones.
Archivo.- 03_flexibilidad.php

Estructurar el código de manera que tenga consecuencias locales y no afecte en otros lugares de nuestro código.
Archivo.- 04_consecuenciasLocales.php

Minimizar la repetición.- Es de gran ayuda para minimizar las consecuencias locales.
Archivo.- 04_consecuenciasLocales.php

Lógica y datos juntos.- Tenerlos juntos hace que el cambio sea local.
Archivo.- 05_logicaDatosJuntos.php

Simetría.- Facial de comunicar y entender, expresa balance y consistencia es intuitiva.
Archivo.- 06_simetria.php

Expresión declarativa.- Expresa nuestro código con la mayor intensión posible. Apuesta por un código mas expresivo de lo que se quiere hacer y no como se va hacer. Crud laravel
